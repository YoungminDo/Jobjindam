# MVP 제작 가이드

## 빠른 시작 패턴 3가지

### 패턴 1: 한 줄 요청
아이디어를 한 문장으로 말하면 자동으로 분석 → 제안 → 구현합니다.

```
"반려동물 산책 매칭 서비스 만들어줘"
```

**처리 흐름:**
1. 핵심 기능 추출 (매칭, 프로필, 채팅, 결제)
2. DB 스키마 제안
3. 화면 구성 제안
4. 확인 후 구현

---

### 패턴 2: 기능 명세 요청
원하는 기능을 구체적으로 나열합니다.

```
"독서 기록 앱 만들어줘. 기능:
- 책 검색 (ISBN API)
- 독서 기록 (날짜, 페이지, 메모)
- 월간 통계
- 독서 목표 설정"
```

**처리 흐름:**
1. 기능별 우선순위 정리
2. 1차 MVP 범위 확정
3. 즉시 구현

---

### 패턴 3: 레퍼런스 기반 요청
비슷한 서비스를 언급하고 차별점을 설명합니다.

```
"당근마켓 같은 중고거래인데, 재능거래 전문으로 만들어줘.
차별점: 포트폴리오 기능, 실시간 견적"
```

**처리 흐름:**
1. 레퍼런스 기능 분석
2. 차별점 반영한 설계
3. 핵심부터 구현

---

## 단계별 프로세스

### Step 1: 아이디어 → 기능 정의 (5분)
```
나: "[아이디어 설명]"
AI: 핵심 기능 분석 결과를 제시
나: 확인 또는 수정 요청
```

### Step 2: 데이터 설계 (5분)
```
AI: Supabase 테이블 스키마 제안
    - 테이블 구조
    - RLS 정책
    - 관계 설정
나: 확인 또는 수정 요청
```

### Step 3: 화면 설계 (5분)
```
AI: 페이지 구성 제안
    - 페이지 목록
    - 각 페이지 주요 컴포넌트
    - 네비게이션 흐름
나: 확인 또는 수정 요청
```

### Step 4: 구현 (자동)
확인 완료 후 자동으로:
1. 프로젝트 구조 생성
2. DB 스키마 SQL 생성
3. 인증 시스템 구현
4. 페이지별 순차 구현
5. 빌드 및 타입 체크

### Step 5: 배포
```
나: "배포해줘" 또는 "Vercel에 올려줘"
AI: 배포 가이드 및 환경변수 설정 안내
```

## 요청 템플릿

### 템플릿 A: 기본 요청
```
[서비스명] 만들어줘.

타겟: [누가 사용하는지]
핵심기능: [가장 중요한 기능 1~3개]
```

### 템플릿 B: 상세 요청
```
[서비스명] 만들어줘.

타겟: [누가 사용하는지]
문제: [어떤 문제를 해결하는지]
핵심기능:
  1. [기능1]
  2. [기능2]
  3. [기능3]
차별점: [기존 서비스와 다른 점]
수익모델: [어떻게 돈을 벌 건지]
```

### 템플릿 C: 클론 + 변형
```
[레퍼런스 서비스] 같은 건데,
[타겟]을 위한 [차별점] 버전으로 만들어줘.

추가기능: [원본에 없는 기능]
제거기능: [원본에서 빼고 싶은 기능]
```

## 자주 쓰는 추가 요청

```
# 기능 추가
"결제 기능 붙여줘" → Supabase + 토스페이먼츠/스트라이프 연동
"알림 기능 추가해줘" → Supabase Realtime 활용
"관리자 페이지 만들어줘" → /admin 라우트 자동 생성
"다크모드 추가해줘" → next-themes + Tailwind dark mode

# 개선
"모바일 최적화해줘" → 반응형 개선
"속도 개선해줘" → 이미지 최적화, 코드 스플리팅
"SEO 적용해줘" → 메타데이터, sitemap, robots.txt
"접근성 개선해줘" → ARIA, 키보드 네비게이션

# 데이터
"더미 데이터 넣어줘" → Supabase seed 데이터
"엑셀 내보내기 추가해줘" → CSV/Excel export
"차트 추가해줘" → Recharts 연동
```
